## Design Principles

<table width="100%">
<tr><td bgcolor="#FFF0F5">

1. ** Human-first** â€” write for the next reader, not just the interpreter

</td></tr>
<tr><td bgcolor="#F0FFF5">

2. ** Small, strong units** â€” short functions with single responsibility

</td></tr>
<tr><td bgcolor="#F0F5FF">

3. ** Reproducible by default** â€” deterministic seeds, pinned environments, clear IO

</td></tr>
<tr><td bgcolor="#FFF5F0">

4. ** Typed and tested** â€” surface failures early, document contracts in code

</td></tr>
<tr><td bgcolor="#F5F0FF">

5. ** Aesthetic clarity** â€” visual defaults that make patterns obvious

</td></tr>
</table>

with types

`list[str]` / `dict`
